# [공모전 서비스 기획서] AI 기반 개발 역량 실시간 관제 솔루션: Dev-Radar

## 0. Executive Summary
- 서비스명: `Dev-Radar` (데브 레이더)
- 슬로건: "추상적인 성장을 확신으로, 데이터 기반 개발자 성장 트래커"
- 한 줄 요약: 학습자의 IDE와 GitHub를 연동하여 코드 품질 및 학습 패턴을 분석하고, 이를 6각형 역량 지표 및 채용 시장 적합도로 시각화하여 학습 불안감을 해소하는 AI 솔루션

## 1. 🎯 문제 정의 (Pain Point)

### 1.1. 학습자의 '성장 안개' 현상
독학이나 부트캠프 수강생들은 매일 코드를 작성하지만, 자신의 실력이 실제 현업 기준에 얼마나 부합하는지 알 수 없어 막연한 불안감을 느낍니다.

### 1.2. 정량적 평가 지표의 부재
기존의 '잔디 심기(GitHub Commit)'는 단순 성실도만 보여줄 뿐, 코드의 질(Quality), 아키텍처 이해도, 보안 준수 여부 등을 증명하지 못합니다.

### 1.3. 시장 요구사항과의 괴리
본인이 학습 중인 기술 스택이 실제 채용 시장의 수요와 얼마나 일치하는지 파악하지 못해 전략적인 취업 준비가 어렵습니다.

## 2. 💡 솔루션 개요

### 2.1. 서비스 컨셉
학습자의 개발 환경을 실시간 관제하여 **개발자 스펙 시각화 대시보드**를 제공합니다.

### 2.2. 핵심 가치
- `Transparency` (투명성): 학습 과정을 데이터로 투명하게 공개
- `Validation` (검증): AI를 통한 객관적 코드 리뷰 및 역량 평가
- `Direction` (방향성): 채용 시장 데이터를 기반으로 한 학습 로드맵 제시

## 3. ⚙ 핵심 기능 (Feature)

| 기능명 | 상세 내용 | AI 역할 |
| --- | --- | --- |
| Live Hexagon Metric | 가독성, 효율성, 보안성 등 6개 지표 시각화 | LLM 기반 코드 정성 평가 및 점수화 |
| Market-Fit Index | 타겟 기업 공고와 내 기술 스택 일치도 산출 | 채용 공고(JD) 자연어 처리 및 유사도 분석 |
| ConceptGap Tracker | 실패한 빌드/오류 분석을 통한 기초 개념 결손 파악 | 오류 로그 분석 및 관련 CS 개념 매핑 |
| AI Peer Review | 커밋 전 실무 관점의 코드 개선안 제안 | 시니어 개발자 코드 패턴 매칭 및 가이드 |

## 4. 🧠 AI 활용 방식

### 4.1. 정성적 코드 평가 알고리즘
- 기술: `LLM(Large Language Model)` + 정적 분석 도구 연동
- 방식: 단순 문법 오류 탐지를 넘어 변수 명명 규칙, 함수의 단일 책임 원칙 준수 여부 등을 분석하여 점수화  
  - 수식: `$Score_{clean} = \sum w_i \cdot c_i$`

### 4.2. 채용 시장 데이터 매칭 (Vector Similarity)
- 기술: `RAG(Retrieval-Augmented Generation)` + 임베딩
- 방식: 최신 채용 공고의 요구 역량 벡터와 학습자 프로젝트 코드 벡터를 비교해 코사인 유사도로 적합도 측정  
  - 수식: `$\cos \theta$`

## 5. 📈 차별화 포인트
- Before: "열심히 하면 언젠간 취업 되겠지"라는 막연한 기대
- After(Dev-Radar): "내 클린 코드 점수가 85점이니, 이제 아키텍처 패턴을 보완하자"라는 데이터 기반 전략
- Competitor 대비: 기존 교육 플랫폼은 '강의 수강 완료율'에 집중하나, 본 서비스는 **실제 구현 역량의 변화**를 실시간 추적

## 6. 💰 비즈니스 모델 (BM)
- B2C(개인): 프리미엄 리포트 및 AI 상세 피드백 (`월 19,000원` 구독)
- B2B(기관): 부트캠프/대학교용 학생 관리 대시보드 (`SaaS` 라이선스)
- B2B(기업): 데이터 기반 인재 추천 및 검증된 포트폴리오 열람 서비스

## 7. 🚀 MVP 및 로드맵

### 7.1. 1단계 (MVP)
- VS Code 확장 프로그램 출시: Python/JS 대상 클린 코드 점수 및 일일 코드량 시각화

### 7.2. 2단계 (고도화)
- GitHub 전체 레포지토리 분석 기능 추가 및 채용 공고 매칭 엔진 탑재

### 7.3. 3단계 (확장)
- 기업 채용 관리 시스템(ATS)과 연동하여 `Dev-Radar 인증서`를 공식 채용 지표로 채택 유도

## 8. 📊 기대 효과
- 학습자 측면: 성장의 가시화를 통한 학습 몰입도 증대 및 취업 성공률 제고
- 교육 기관 측면: 학생별 맞춤형 관리가 가능해져 중도 탈락률 30% 이상 감소 기대
- 기업 측면: 허위 포트폴리오 필터링 및 실무 역량 검증 인재 확보 비용 절감
